# Packaging

This project can be packaged as a Windows desktop app with PyInstaller.

## Build Requirements

- Windows 10 or newer
- Python 3.11 or newer
- PowerShell
- Internet access for the first dependency install

## Build The App

From the project root:

```powershell
.\scripts\build_windows.ps1 -Clean
```

The packaged app will be created at:

```text
dist\LocalTranscriber\LocalTranscriber.exe
```

Share the full `dist\LocalTranscriber` folder, not only the `.exe`. PyInstaller's
folder build keeps supporting DLLs and Python packages beside the executable.

## Build With GitHub Actions

The repository includes `.github/workflows/windows-build.yml`.

You can run it manually from the GitHub Actions tab, or create a tag like
`v0.1.0` and push it. The workflow uploads a `LocalTranscriber-windows`
artifact containing the packaged app folder.

## First Launch Notes

The app downloads the selected Whisper model from Hugging Face the first time it
is used. After that, model files are cached locally by `huggingface_hub`.

Media files are processed locally. The app does not upload audio or video to a
transcription service.

## Troubleshooting

If the build fails because `python` is not recognized, install Python from
python.org and enable `Add python.exe to PATH`.

If Windows SmartScreen warns about the app, that is expected for unsigned local
builds. A public release can reduce this with code signing.

If startup is slow, use the folder build generated by the script. A single-file
PyInstaller build is possible, but usually starts more slowly.
